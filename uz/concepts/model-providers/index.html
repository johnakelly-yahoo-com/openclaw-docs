    49. Model provayderlari¶

    1. Bu sahifa LLM/model provayderlarini qamrab oladi (WhatsApp/Telegram kabi chat kanallari emas).
    2. Modelni tanlash qoidalari uchun qarang: /concepts/models.

    2. Tezkor qoidalar¶

      1. Model havolalari provider/model formatidan foydalanadi (misol: opencode/claude-opus-4-6).
      1. Agar agents.defaults.models ni o‘rnatsangiz, u ruxsat etilgan ro‘yxatga (allowlist) aylanadi.
      1. CLI yordamchilari: openclaw onboard, openclaw models list, openclaw models set <provider/model>.

    6. Ichki provayderlar (pi‑ai katalogi)¶

    1. OpenClaw pi‑ai katalogi bilan birga yetkazib beriladi. 8. Ushbu provayderlar hech qanday models.providers konfiguratsiyasini talab qilmaydi; faqat autentifikatsiyani sozlab, modelni tanlang.

    9. OpenAI¶

      1. Provayder: openai
      1. Autentifikatsiya: OPENAI_API_KEY
      1. Namuna model: openai/gpt-5.1-codex
      1. CLI: openclaw onboard --auth-choice openai-api-key
    14. {
  agents: { defaults: { model: { primary: "openai/gpt-5.1-codex" } } },
}

    15. Anthropic¶

      1. Provayder: anthropic
      1. Autentifikatsiya: ANTHROPIC_API_KEY yoki claude setup-token
      1. Namuna model: anthropic/claude-opus-4-6
      1. CLI: openclaw onboard --auth-choice token (setup-tokenni joylashtiring) yoki openclaw models auth paste-token --provider anthropic
    20. {
  agents: { defaults: { model: { primary: "anthropic/claude-opus-4-6" } } },
}

    21. OpenAI Code (Codex)¶

      1. Provayder: openai-codex
      1. Autentifikatsiya: OAuth (ChatGPT)
      1. Namuna model: openai-codex/gpt-5.3-codex
      1. CLI: openclaw onboard --auth-choice openai-codex yoki openclaw models auth login --provider openai-codex
    26. {
  agents: { defaults: { model: { primary: "openai-codex/gpt-5.3-codex" } } },
}

    27. OpenCode Zen¶

      1. Provayder: opencode
      1. Autentifikatsiya: OPENCODE_API_KEY (yoki OPENCODE_ZEN_API_KEY)
      1. Namuna model: opencode/claude-opus-4-6
      1. CLI: openclaw onboard --auth-choice opencode-zen
    32. {
  agents: { defaults: { model: { primary: "opencode/claude-opus-4-6" } } },

    33. Google Gemini (API kalit)¶

      1. Provayder: google
      1. Autentifikatsiya: GEMINI_API_KEY
      1. Namuna model: google/gemini-3-pro-preview
      1. CLI: openclaw onboard --auth-choice gemini-api-key

    38. Google Vertex, Antigravity va Gemini CLI¶

      1. Provayderlar: google-vertex, google-antigravity, google-gemini-cli
      1. Autentifikatsiya: Vertex gcloud ADC’dan foydalanadi; Antigravity/Gemini CLI esa o‘ziga xos autentifikatsiya jarayonlaridan foydalanadi
      1. Antigravity OAuth biriktirilgan plagin sifatida yetkazib beriladi (google-antigravity-auth, sukut bo‘yicha o‘chirilgan).
      1. Yoqish: openclaw plugins enable google-antigravity-auth
      1. Kirish: openclaw models auth login --provider google-antigravity --set-default
      1. Gemini CLI OAuth biriktirilgan plagin sifatida yetkazib beriladi (google-gemini-cli-auth, sukut bo‘yicha o‘chirilgan).
      1. Yoqish: openclaw plugins enable google-gemini-cli-auth
      1. Kirish: openclaw models auth login --provider google-gemini-cli --set-default
      1. Eslatma: openclaw.json ichiga mijoz identifikatori (client id) yoki maxfiy kalitni (secret) kiritmaysiz. 48. CLI orqali kirish jarayoni tokenlarni shlyuz xostidagi autentifikatsiya profillarida saqlaydi.

    22. Moonshot AI (Kimi)¶

    1. Moonshot OpenAI-mos endpointlardan foydalanadi, shuning uchun uni maxsus provayder sifatida sozlang:

      1. Provayder: moonshot
      1. Avtorizatsiya: MOONSHOT_API_KEY
      1. Namunaviy model: moonshot/kimi-k2.5

    5. Kimi K2 model IDlari:

    6. {/moonshot-kimi-k2-model-refs:start/ && null}

      1. moonshot/kimi-k2.5
      1. moonshot/kimi-k2-0905-preview
      1. moonshot/kimi-k2-turbo-preview
      1. moonshot/kimi-k2-thinking

    MiniMax¶

    MiniMax is configured via models.providers because it uses custom endpoints:

    • MiniMax (Anthropic‑compatible): --auth-choice minimax-api
    • Auth: MINIMAX_API_KEY

    See /providers/minimax for setup details, model options, and config snippets.

    Ollama¶

    Ollama is a local LLM runtime that provides an OpenAI-compatible API:

    • Provider: ollama
    • Auth: None required (local server)
    • Example model: ollama/llama3.3
    • Installation: https://ollama.ai
    # Install Ollama, then pull a model:
ollama pull llama3.3

    {
  agents: {
    defaults: { model: { primary: "ollama/llama3.3" } },
  },
}

    Ollama is automatically detected when running locally at http://127.0.0.1:11434/v1. See /providers/ollama for model recommendations and custom configuration.

    Local proxies (LM Studio, vLLM, LiteLLM, etc.)¶

    Example (OpenAI‑compatible):

    {
  agents: {
    defaults: {
      model: { primary: "lmstudio/minimax-m2.1-gs32" },
      models: { "lmstudio/minimax-m2.1-gs32": { alias: "Minimax" } },
    },
  },
  models: {
    providers: {
      lmstudio: {
        baseUrl: "http://localhost:1234/v1",
        apiKey: "LMSTUDIO_KEY",
        api: "openai-completions",
        models: [
          {
            id: "minimax-m2.1-gs32",
            name: "MiniMax M2.1",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 200000,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}

    Notes:

    • For custom providers, reasoning, input, cost, contextWindow, and maxTokens are optional. When omitted, OpenClaw defaults to:
    • reasoning: false
    • input: ["text"]
    • cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 }
    • contextWindow: 200000
    • maxTokens: 8192
    • Recommended: set explicit values that match your proxy/model limits.
