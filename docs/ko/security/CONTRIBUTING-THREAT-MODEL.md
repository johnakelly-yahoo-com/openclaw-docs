# 26. OpenClaw 위협 모델에 기여하기

27. OpenClaw를 더 안전하게 만드는 데 도움을 주셔서 감사합니다. 28. 이 위협 모델은 살아있는 문서이며, 보안 전문가가 아니어도 누구나 기여를 환영합니다.

## 29. 기여 방법

### 30. 위협 추가하기

31. 우리가 다루지 않은 공격 벡터나 위험을 발견하셨나요? 32. [openclaw/trust](https://github.com/openclaw/trust/issues)에 이슈를 열고 자신의 말로 설명해 주세요. 33. 어떤 프레임워크를 알 필요도, 모든 필드를 채울 필요도 없습니다. 시나리오만 설명하면 됩니다.

34. **포함하면 도움이 되는 사항(필수 아님):**

- 35. 공격 시나리오와 악용 가능 방식
- 36. 영향을 받는 OpenClaw 구성 요소(CLI, 게이트웨이, 채널, ClawHub, MCP 서버 등)
- 37. 심각도에 대한 의견(낮음 / 중간 / 높음 / 치명적)
- 38. 관련 연구, CVE 또는 실제 사례에 대한 링크

39. ATLAS 매핑, 위협 ID 및 위험 평가는 검토 과정에서 우리가 처리합니다. 40. 해당 세부 정보를 포함하고 싶다면 좋지만, 기대 사항은 아닙니다.

> 41. **이는 위협 모델에 추가하기 위한 것이며, 실제 취약점 보고가 아닙니다.** 악용 가능한 취약점을 발견한 경우 책임 있는 공개 절차는 [Trust 페이지](https://trust.openclaw.ai)를 참조하세요.

### 42. 완화책 제안하기

43. 기존 위협을 해결할 아이디어가 있나요? 44. 해당 위협을 참조하는 이슈 또는 PR을 열어 주세요. 45. 유용한 완화책은 구체적이고 실행 가능해야 합니다. 예를 들어 "게이트웨이에서 발신자별 분당 10건의 속도 제한"은 "속도 제한 구현"보다 낫습니다.

### 46. 공격 체인 제안하기

47. 공격 체인은 여러 위협이 결합되어 현실적인 공격 시나리오가 되는 과정을 보여줍니다. 48. 위험한 조합을 발견했다면, 공격자가 어떻게 단계적으로 연결해 실행하는지 설명해 주세요. 49. 형식적인 템플릿보다 실제로 공격이 전개되는 짧은 서술이 더 가치 있습니다.

### 50. 기존 콘텐츠 수정 또는 개선하기

오타, 설명 보완, 오래된 정보, 더 나은 예시 등 - PR 환영, 이슈 등록은 필요 없습니다.

## 우리가 사용하는 것

### MITRE ATLAS

이 위협 모델은 [MITRE ATLAS](https://atlas.mitre.org/) (AI 시스템을 위한 적대적 위협 환경, Adversarial Threat Landscape for AI Systems)를 기반으로 하며, 프롬프트 인젝션, 도구 오용, 에이전트 악용과 같은 AI/ML 위협을 위해 특별히 설계된 프레임워크입니다. 기여하기 위해 ATLAS를 알 필요는 없습니다 - 제출된 내용은 검토 과정에서 프레임워크에 매핑됩니다.

### 위협 ID

각 위협에는 `T-EXEC-003`과 같은 ID가 부여됩니다. 카테고리는 다음과 같습니다:

| 코드      | 카테고리            |
| ------- | --------------- |
| RECON   | 정찰 - 정보 수집      |
| ACCESS  | 초기 접근 - 진입 획득   |
| EXEC    | 실행 - 악의적인 동작 실행 |
| PERSIST | 지속성 - 접근 유지     |
| EVADE   | 방어 회피 - 탐지 회피   |
| DISC    | 발견 - 환경 파악      |
| EXFIL   | 유출 - 데이터 탈취     |
| IMPACT  | 영향 - 피해 또는 중단   |

ID는 검토 과정에서 관리자가 할당합니다. 직접 선택할 필요는 없습니다.

### 위험 수준

| 수준      | 의미                                 |
| ------- | ---------------------------------- |
| **치명적** | 전체 시스템 장악, 또는 높은 발생 가능성 + 치명적인 영향  |
| **높음**  | 상당한 피해 가능성, 또는 중간 발생 가능성 + 치명적인 영향 |
| **중간**  | 보통 수준의 위험, 또는 낮은 발생 가능성 + 높은 영향    |
| **낮음**  | 발생 가능성이 낮고 영향이 제한적임                |

위험 수준이 확실하지 않다면, 영향만 설명해 주세요. 저희가 평가하겠습니다.

## 검토 프로세스

1. **분류(Triage)** - 신규 제출물은 48시간 이내에 검토합니다.
2. **평가(Assessment)** - 실행 가능성을 검증하고, ATLAS 매핑 및 위협 ID를 할당하며, 위험 수준을 검증합니다.
3. **문서화(Documentation)** - 모든 내용이 형식에 맞고 완전한지 확인합니다.
4. **병합(Merge)** - 위협 모델과 시각화에 추가됩니다.
