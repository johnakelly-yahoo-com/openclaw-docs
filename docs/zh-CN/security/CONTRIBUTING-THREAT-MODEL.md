# 为 OpenClaw 威胁模型做出贡献

感谢你帮助让 OpenClaw 更加安全。 该威胁模型是一个持续演进的文档，我们欢迎任何人的贡献——你不需要是安全专家。

## 贡献方式

### 添加一个威胁

发现了我们尚未覆盖的攻击向量或风险？ 请在 [openclaw/trust](https://github.com/openclaw/trust/issues) 上提交一个 issue，并用你自己的话进行描述。 你不需要了解任何框架或填写所有字段——只需描述场景即可。

**建议包含（但非必须）：**

- 攻击场景以及它可能如何被利用
- 受影响的 OpenClaw 组件（CLI、gateway、channels、ClawHub、MCP servers 等）
- 你认为其严重程度（低 / 中 / 高 / 严重）
- 任何相关研究、CVE 或真实世界示例的链接

1. 我们将在评审期间处理 ATLAS 映射、威胁 ID 和风险评估。 2. 如果你想包含这些细节，很好——但这不是必需的。

> 3. **这是用于添加到威胁模型中，而不是报告正在发生的漏洞。** 如果你发现了可被利用的漏洞，请参阅我们的 [Trust 页面](https://trust.openclaw.ai) 了解负责任披露的说明。

### Suggest a Mitigation

5. 是否有想法来应对一个现有威胁？ 6. 提交一个引用该威胁的 issue 或 PR。 7. 有用的缓解措施应当具体且可执行——例如，“在网关处对每个发送者实施 10 条消息/分钟的速率限制”要优于“实现速率限制”。

### 8. 提出一个攻击链

9. 攻击链展示了多个威胁如何组合成一个现实的攻击场景。 10. 如果你看到了一个危险的组合，请描述步骤以及攻击者将如何把它们串联起来。 11. 相比正式模板，对攻击在实践中如何展开的简短叙述更有价值。

### 12. 修复或改进现有内容

13. 拼写错误、澄清、过时信息、更好的示例——欢迎 PR，无需提交 issue。

## 14. 我们使用的内容

### 15. MITRE ATLAS

1. 此威胁模型基于 [MITRE ATLAS](https://atlas.mitre.org/)（面向 AI 系统的对抗性威胁全景），这是一个专门为提示注入、工具滥用和代理利用等 AI/ML 威胁而设计的框架。 17. 你不需要了解 ATLAS 也可以贡献——我们会在评审期间将提交内容映射到该框架。

### 18. 威胁 ID

19. 每个威胁都会获得一个类似 `T-EXEC-003` 的 ID。 20. 分类包括：

| 21. 代码      | 22. 分类           |
| ---------------------------------- | --------------------------------------- |
| 23. RECON   | 24. 侦察——信息收集     |
| 25. ACCESS  | 26. 初始访问——获得进入权限 |
| 27. EXEC    | 28. 执行——运行恶意操作   |
| 29. PERSIST | 30. 持久化——维持访问    |
| 31. EVADE   | 32. 规避防御——避免被检测  |
| 33. DISC    | 34. 发现——了解环境     |
| 35. EXFIL   | 36. 外泄——窃取数据     |
| 37. IMPACT  | 38. 影响——造成损害或中断  |

39. ID 由维护者在评审期间分配。 40. 你不需要自行选择。

### 41. 风险等级

| 42. 等级     | 43. 含义                    |
| --------------------------------- | ------------------------------------------------ |
| 44. **严重** | 45. 完全系统被攻陷，或发生概率高且影响严重   |
| 46. **高**  | 47. 可能造成重大损害，或发生概率中等且影响严重 |
| 48. **中**  | 49. 中等风险，或发生概率低但影响高       |
| 50. **低**  | 不太可能且影响有限                                        |

如果你不确定风险级别，只需描述影响，我们会进行评估。

## 审查流程

1. **初步筛选** - 我们会在 48 小时内审查新的提交
2. 2. **评估** - 我们验证可行性，分配 ATLAS 映射和威胁 ID，并验证风险级别
3. **文档整理** - 我们确保所有内容格式正确且完整
4. **合并** - 添加到威胁模型和可视化中
